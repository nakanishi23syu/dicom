# 環境構築手順書

このドキュメントは `dicom-tool` の開発環境をゼロから構築する手順です。
同じ構成（Vue 3 フロント + C#/GraphQL バック のモノレポ）で別プロジェクトを立ち上げる際のテンプレートとしても使えるように、
「何を」「なぜ」入れるかを併記しています。

## 1. 全体構成

```
<project-root>/
├── frontend/     … Vue 3 + TypeScript + Vite
├── backend/      … C# + ASP.NET Core + HotChocolate(GraphQL) + Entity Framework Core + SQLite
├── .claude/      … Claude Code 設定（今後追加）
├── 環境構築手順書.md
└── README.md
```

フロントエンドとバックエンドを1つのGitリポジトリ配下で `frontend/` `backend/` に分け、
Claude Code の設定やドキュメント類はリポジトリ直下（親フォルダ）にまとめる、モノレポ構成を採用しています。
理由: 学習用途では別リポジトリに分けるとPR連携やバージョン管理が煩雑になるため、まずは1リポジトリで完結させる。

## 2. 前提ソフトウェア

| ソフトウェア | 用途 | 確認コマンド | このプロジェクトでの動作確認済みバージョン |
|---|---|---|---|
| [Node.js](https://nodejs.org/) | フロントエンドのビルド・実行 | `node --version` | v24.13.0 |
| npm | パッケージ管理（Node.js に同梱） | `npm --version` | 11.6.2 |
| [.NET SDK](https://dotnet.microsoft.com/download) | バックエンドのビルド・実行 | `dotnet --version` | 10.0.204（9.0系でも動作可） |
| [Git](https://git-scm.com/) | バージョン管理 | `git --version` | - |
| [Visual Studio](https://visualstudio.microsoft.com/) (Community可) | バックエンド(C#)の開発 | - | 2022以降、ワークロード「ASP.NET と Web 開発」を選択 |
| VS Code | フロントエンド(Vue/TS)の開発 | - | 拡張機能 `Vue - Official`、`ESLint`、`Prettier` を推奨 |

バージョンは目安です。.NET SDK は `backend/**/*.csproj` の `<TargetFramework>` に合わせて選んでください。

## 3. リポジトリ取得

```bash
git clone <repository-url>
cd dicom-tool
```

## 4. フロントエンド環境構築

```bash
cd frontend
npm install
npm run dev
```

- 起動後、ブラウザで `http://localhost:5173` を開く。
- Lint/Format:
  ```bash
  npm run lint       # ESLintチェック
  npm run lint:fix    # 自動修正
  npm run format      # Prettierで整形
  ```
- ビルド: `npm run build`（`dist/` に静的ファイルを出力）

### DICOMサンプルデータの用意

`public/dicom/` にサンプル `.dcm` ファイルが必要です。TCIAからダウンロードする場合:

```bash
cd frontend
pip install requests
python download_dicom.py
```

詳細は [`frontend/README.md`](./frontend/README.md) を参照。

## 5. バックエンド環境構築

### 5-1. Visual Studio を使う場合（推奨）

1. Visual Studio を起動し、`backend/DicomLearning.GraphQL.slnx` を開く。
2. ソリューションエクスプローラーで起動プロジェクトが `DicomLearning.GraphQL` になっていることを確認。
3. `F5`（デバッグ実行）または `Ctrl+F5`（デバッグなし実行）で起動。
4. 起動時のコンソールに表示されるURL（例: `http://localhost:5030`）を確認。

### 5-2. コマンドラインを使う場合

```bash
cd backend/DicomLearning.GraphQL
dotnet restore
dotnet run
```

### 5-3. 動作確認（GraphQL IDE）

ブラウザで `http://localhost:<ポート番号>/graphql` を開くと、GraphQL用のIDEが起動します。

- 既定では **Nitro**（HotChocolateの標準GraphQL IDE。旧BananaCakePopの後継）が開きます。
- **BananaCakePop** を使いたい場合は `Program.cs` の `AddGraphQLServer()` の設定で `.ModifyRequestOptions(...)` や `.AddBananaCakePop()`（Nitroへの統合以前の旧パッケージ相当の設定）を利用します。プロジェクトの設定を変更する場合はチーム内で相談してください。

curlで直接叩く場合:

```bash
curl -X POST http://localhost:5030/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"query { studies { patientName studyDate } }"}'
```

### 5-4. データベース（EF Core + SQLite）※導入後

EF Core Migration を追加した後は、以下のコマンドでDBを作成・更新します（`dotnet-ef` ツールが必要）。

```bash
dotnet tool install --global dotnet-ef   # 初回のみ
cd backend/DicomLearning.GraphQL
dotnet ef database update
```

詳細は [`backend/README.md`](./backend/README.md) を参照。

## 6. .gitignore の方針

- `node_modules/`, `dist/`, `backend/**/bin/`, `backend/**/obj/` … 再生成可能なビルド成果物・依存パッケージは除外。
- `.env`, `.env.*.local` … ローカル環境固有の秘匿情報は除外（`.env.example` 等のテンプレートはコミット対象にする）。
- `.vscode/`, `.idea/` … エディタ固有設定は除外（個人の好みに依存するため）。
- `*.sln` … Visual Studioのレガシー形式は除外対象だが、本プロジェクトは新形式の `*.slnx` を使用しておりこちらはコミット対象。

## 7. このドキュメントを他プロジェクトに流用する場合

1. 「2. 前提ソフトウェア」の表をプロジェクトの実際の技術スタックに合わせて書き換える。
2. 「1. 全体構成」のディレクトリツリーを実際の構成に合わせる。
3. フロント/バックそれぞれの詳細セットアップ手順は各ディレクトリの `README.md` に書き、このドキュメントからはリンクするだけにする（手順の二重管理を避けるため）。
